# 图片路径问题修复

## 问题描述

在部署到 GitHub Pages 后，物品图片无法显示。

---

## 原因分析

### 1. Base 路径配置
GitHub Pages 部署在子路径 `/DarkSouls-Tome/` 下，需要正确配置 base 路径。

### 2. 图片路径
图片存放在 `public/icons/` 目录下，在生产环境中需要加上 base 路径前缀。

---

## 解决方案

### 1. Vite 配置（已完成）

```typescript
// vite.config.ts
export default defineConfig({
  base: process.env.NODE_ENV === 'production' ? '/DarkSouls-Tome/' : '/',
  // ...
})
```

### 2. 图片路径使用

在组件中使用图片时，直接使用 `/icons/` 开头的路径，Vite 会自动添加 base 前缀：

```vue
<!-- 正确 ✅ -->
<img :src="`/icons/${item.icon}`" :alt="item.name.chn">

<!-- 错误 ❌ -->
<img :src="`icons/${item.icon}`" :alt="item.name.chn">
<img :src="`./icons/${item.icon}`" :alt="item.name.chn">
```

### 3. 懒加载图片

LazyImage 组件已正确配置：

```vue
<LazyImage :src="`/icons/${item.icon}`" :alt="displayName(item)" />
```

## TypeScript 错误修复

### 问题 1: LazyImage.vue imgElement 未使用

**错误信息**:
```
error TS6133: 'imgElement' is declared but its value is never read.
```

**原因**:
在 Vue 3 的 `<script setup>` 中，template ref 是自动绑定的，TypeScript 检测不到这种隐式使用。

**解决方案**:
```typescript
// @ts-expect-error imgElement is used via template ref
const { src, isLoaded, error, imgElement } = useLazyImage(props.src, props.placeholder);
```

### 问题 2: Home.vue GameVersion 类型错误（已解决）

这个错误可能是 GitHub Actions 缓存了旧代码导致的。当前 Home.vue 已经不使用 GameVersion 类型。

## 常见问题

### Q1: 图片路径 404

**检查项**:
1. 图片文件是否存在于 `public/icons/` 目录
2. 图片路径是否以 `/` 开头
3. base 路径配置是否正确
4. 构建后的 dist 目录是否包含 icons 文件夹

**解决方案**:
```bash
# 检查图片文件
ls public/icons/

# 检查构建输出
npm run build
ls dist/icons/
```

### Q2: 开发环境正常，生产环境图片不显示

**原因**: base 路径配置问题

**解决方案**:
```typescript
// vite.config.ts
base: process.env.NODE_ENV === 'production' ? '/DarkSouls-Tome/' : '/'
```

### Q3: GitHub Actions 构建失败

**可能原因**:
1. TypeScript 类型错误
2. 缓存了旧代码
3. 依赖版本不兼容

**解决方案**:
```yaml
# .github/workflows/deploy.yml
- name: Checkout
  uses: actions/checkout@v4
  with:
    clean: true  # 清理工作目录

- name: Install dependencies
  run: npm ci  # 使用 ci 而不是 install，确保依赖一致
```

## 图片优化建议

### 1. 图片格式
当前使用 WebP 格式，已经是最优选择。

### 2. 图片大小
建议图标尺寸：
- 列表图标: 48x48px
- 详情大图标: 128x128px
- 相关推荐: 64x64px

### 3. 懒加载
已实现 IntersectionObserver 懒加载：
- 提前 50px 开始加载
- 不支持的浏览器降级为直接加载

### 4. 缓存策略
PWA 配置了图片缓存：
```typescript
runtimeCaching: [
  {
    urlPattern: /^https:\/\/.*\.webp$/,
    handler: 'CacheFirst',
    options: {
      cacheName: 'images-cache',
      expiration: {
        maxEntries: 1000,
        maxAgeSeconds: 60 * 60 * 24 * 30 // 30天
      }
    }
  }
]
```
