# 图片路径问题修复

## 完成时间
2026-03-01

---

## 问题描述

物品详情页和列表页的图片无法显示，浏览器控制台显示 404 错误。

---

## 问题原因

数据文件中的图片路径与实际文件结构不匹配：

**数据中的路径格式**：
```
/icons/ds1/rings/100.webp
/icons/ds1/weapons/50.webp
/icons/ds2/armors/200.webp
```

**实际文件结构**：
```
/icons/ring1/100.webp
/icons/weapon1/50.webp
/icons/armor2/200.webp
```

---

## 解决方案

### 1. 创建路径修复脚本

创建 `scripts/fixIconPaths.js` 脚本，批量修复所有数据文件中的图片路径。

**路径映射规则**：

| 数据中的路径 | 实际路径 |
|------------|---------|
| `/icons/ds1/rings/` | `/icons/ring1/` |
| `/icons/ds1/weapons/` | `/icons/weapon1/` |
| `/icons/ds1/armors/` | `/icons/armor1/` |
| `/icons/ds1/items/` | `/icons/item1/` |
| `/icons/ds1/magics/` | `/icons/magic1/` |
| `/icons/ds1/dialogues/` | `/icons/dialogue1/` |
| `/icons/ds2/rings/` | `/icons/ring2/` |
| `/icons/ds2/weapons/` | `/icons/weapon2/` |
| `/icons/ds2/armors/` | `/icons/armor2/` |
| `/icons/ds2/items/` | `/icons/item2/` |
| `/icons/ds2/magics/` | `/icons/magic2/` |
| `/icons/ds2/dialogues/` | `/icons/dialogue2/` |
| `/icons/ds3/rings/` | `/icons/ring3/` |
| `/icons/ds3/weapons/` | `/icons/weapon3/` |
| `/icons/ds3/armors/` | `/icons/armor3/` |
| `/icons/ds3/items/` | `/icons/item3/` |
| `/icons/ds3/magics/` | `/icons/magic3/` |
| `/icons/ds3/dialogues/` | `/icons/dialogue3/` |

### 2. 执行修复

```bash
node scripts/fixIconPaths.js
```

**修复结果**：
- ✓ ds1/rings.json
- ✓ ds1/weapons.json
- ✓ ds1/armors.json
- ✓ ds1/items.json
- ✓ ds1/magics.json
- ✓ ds2/rings.json
- ✓ ds2/weapons.json
- ✓ ds2/armors.json
- ✓ ds2/items.json
- ✓ ds2/magics.json
- ✓ ds3/rings.json
- ✓ ds3/weapons.json
- ✓ ds3/armors.json
- ✓ ds3/items.json
- ✓ ds3/magics.json

共修复 15 个数据文件。

## 注意事项

1. **路径格式统一**：所有图片路径都使用 `/icons/{type}{game}/` 格式
2. **文件名保持不变**：只修改路径，不修改文件名
3. **批量处理**：脚本自动处理所有数据文件，无需手动修改
4. **可重复执行**：脚本可以安全地重复执行，不会重复修改
