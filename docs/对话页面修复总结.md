# 对话页面修复总结

## 问题描述

用户反馈对话页面（如 `/ds1/dialogue/alvina`）只显示 NPC 头像和名字，没有显示任何对话内容。

## 问题原因

1. **数据转换脚本问题**：对话 XML 文件的格式与物品 XML 不同
   - 物品 XML：`<text id="100">内容</text>`（有 id 属性）
   - 对话 XML：`<text>内容</text>` 或 `<text id="48000000">内容</text>`（部分有 id）
   - 原脚本假设所有 text 元素都有 id 属性，导致解析失败

2. **对话数据为空**：由于解析失败，所有对话 JSON 文件的 `lines` 数组都是空的

3. **图标路径不匹配**：对话图标需要从 `reference/icons/dialogue{1,2,3}/` 复制到 `public/icons/ds{1,2,3}/dialogues/`

## 解决方案

### 1. 修复 XML 转换脚本

创建专门的 `parseDialogueXml` 函数处理对话数据：

```javascript
async function parseDialogueXml(filePath) {
  try {
    const xmlData = fs.readFileSync(filePath, 'utf-8');
    const result = await parser.parseStringPromise(xmlData);
    return result.entries.text.map((item, index) => {
      const text = typeof item === 'string' ? item : (item._ || item);
      const id = item.$?.id;
      return {
        index,
        id,
        text
      };
    });
  } catch (error) {
    console.error(`Error parsing ${filePath}:`, error.message);
    return [];
  }
}
```

### 2. 更新对话数据结构

对话数据现在包含完整的三语内容：

```json
{
  "npc": "alvina",
  "game": 1,
  "avatar": "/icons/ds1/dialogues/alvina.webp",
  "lines": [
    {
      "index": 0,
      "chn": "#0第1次攀谈",
      "jap": "#0話しかける1回目",
      "eng": "#0Talk-1st time",
      "isUnused": false,
      "isTitle": true
    },
    {
      "index": 1,
      "chn": "喔，没见过你耶。",
      "jap": "ほぅ、あんた、新顔だね",
      "eng": "Is it not so that thou art new.",
      "isUnused": false,
      "isTitle": false
    }
  ]
}
```

### 3. 应用原版风格

更新 `DialogueCard.vue` 组件样式：

- 深灰背景 `#111` + 内嵌阴影 `#531`
- 金色标题 `#960`
- 仿宋字体
- 标题行高亮显示（金黄色 `#fe6`）
- 废弃对话半透明显示
- 移动端响应式布局

### 4. 复制对话图标

```bash
# DS1
Copy-Item "reference/icons/dialogue1/*" "public/icons/ds1/dialogues/"

# DS2
Copy-Item "reference/icons/dialogue2/*" "public/icons/ds2/dialogues/"

# DS3
Copy-Item "reference/icons/dialogue3/*" "public/icons/ds3/dialogues/"
```

### 5. 调整 PWA 配置

增加缓存文件大小限制以支持大型对话数据文件：

```typescript
workbox: {
  maximumFileSizeToCacheInBytes: 3 * 1024 * 1024, // 3MB
}
```

## 技术要点

1. **XML 解析灵活性**：处理有无 id 属性的不同情况
2. **数据索引**：使用数组索引确保三语对话行对齐
3. **特殊标记识别**：
   - `#0` 开头：标题行（`isTitle: true`）
   - `##` 包含：废弃对话（`isUnused: true`）
4. **原版风格复刻**：参考原项目的配色和布局
5. **全局语言切换**：配合 `userStore.currentLanguage` 只显示当前语言
