## 重构方案

### 方案 A：扁平化结构（推荐）

保持简单的扁平结构，只保留正在使用的组件：

```
src/components/
├── ClassicNavigation.vue      # 顶部导航
├── ClassicContainer.vue       # 容器
├── ClassicItemCard.vue        # 物品卡片
├── SidebarNav.vue            # 侧边栏
├── ItemCatalogTable.vue      # 一览表
├── DialogueCard.vue          # 对话卡片
└── LazyImage.vue             # 懒加载图片
```

**优点**：
- 简单直观
- 易于查找
- 组件数量少（7个）

**缺点**：
- 所有组件在同一层级

---

### 方案 B：分类结构

按功能分类组织组件：

```
src/components/
├── layout/                    # 布局组件
│   ├── ClassicNavigation.vue
│   ├── ClassicContainer.vue
│   └── SidebarNav.vue
├── item/                      # 物品相关
│   ├── ClassicItemCard.vue
│   └── ItemCatalogTable.vue
├── dialogue/                  # 对话相关
│   └── DialogueCard.vue
└── common/                    # 通用组件
    └── LazyImage.vue
```

**优点**：
- 分类清晰
- 易于扩展
- 职责明确

**缺点**：
- 目录层级增加
- 导入路径变长

---

### 方案 C：混合结构

常用组件扁平化，特定功能分类：

```
src/components/
├── ClassicNavigation.vue      # 常用布局
├── ClassicContainer.vue
├── SidebarNav.vue
├── LazyImage.vue             # 常用工具
├── item/                      # 物品功能
│   ├── ClassicItemCard.vue
│   └── ItemCatalogTable.vue
└── dialogue/                  # 对话功能
    └── DialogueCard.vue
```

**优点**：
- 平衡简单性和组织性
- 常用组件易访问
- 功能组件分类

**缺点**：
- 规则不够统一

---

## 推荐方案：方案 A（扁平化）

### 理由
1. **组件数量少**：只有 7 个组件，不需要复杂的目录结构
2. **查找方便**：所有组件在同一目录，一目了然
3. **导入简单**：路径短，易于维护
4. **符合现状**：当前就是扁平结构

### 实施步骤

#### 1. 删除废弃组件
```bash
# 删除现代风格组件
rm src/components/AppNavigation.vue
rm src/components/ItemCard.vue
rm src/components/ItemList.vue
rm src/components/ItemDetailModal.vue
rm src/components/AdvancedSearch.vue
rm src/components/SearchBar.vue
rm src/components/ThemeSwitch.vue
rm src/components/SkeletonCard.vue
rm src/components/LanguageSwitch.vue

# 删除示例文件
rm src/components/HelloWorld.vue

# 删除空目录
rm -rf src/components/common
rm -rf src/components/dialogue
rm -rf src/components/item
```

#### 2. 保留的组件
- ✅ ClassicNavigation.vue
- ✅ ClassicContainer.vue
- ✅ ClassicItemCard.vue
- ✅ SidebarNav.vue
- ✅ ItemCatalogTable.vue
- ✅ DialogueCard.vue
- ✅ LazyImage.vue

#### 3. 检查组件使用情况

需要检查 DialogueCard.vue 是否还在使用：
```bash
grep -r "DialogueCard" src/views/
```

如果 DialogueView.vue 还在使用 DialogueCard，则保留；否则删除。

## 组件职责说明

   - 搜索功能

6. **DialogueCard.vue**

   - 用于 DialogueView
