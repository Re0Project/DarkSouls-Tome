## 修复的问题

### 1. TypeScript 错误：LazyImage.vue ✅

**错误信息**:
```
error TS6133: 'imgElement' is declared but its value is never read.
```

**修复方案**:
```typescript
// src/components/LazyImage.vue
// @ts-expect-error imgElement is used via template ref
const { src, isLoaded, error, imgElement } = useLazyImage(props.src, props.placeholder);
```

**原因**: 在 Vue 3 `<script setup>` 中，template ref 是隐式绑定的，TypeScript 无法检测到这种使用。

## 技术细节

### TypeScript 配置
```json
// tsconfig.app.json
{
  "compilerOptions": {
    "strict": true,
    "noUnusedLocals": true,
    "noUnusedParameters": true
  }
}
```

### Vue 3 Template Ref
在 `<script setup>` 中，template ref 通过变量名自动绑定：
```vue
<template>
  <img ref="imgElement" />
</template>

<script setup>
// imgElement 自动绑定到 template 中的 ref
const imgElement = ref(null)
</script>
```

### Vite 静态资源处理
- `public/` 目录下的文件会被复制到 dist 根目录
- 使用绝对路径 `/icons/` 访问
- Vite 自动添加 base 前缀
