# 部署指南

## GitHub Pages 部署

本项目配置了自动部署到 GitHub Pages 的 GitHub Actions 工作流。

### 前置要求

1. GitHub 仓库已创建
2. 代码已推送到 `main` 分支
3. GitHub Pages 已启用

### 启用 GitHub Pages

1. 进入仓库设置 (Settings)
2. 找到 "Pages" 选项
3. Source 选择 "GitHub Actions"
4. 保存设置

### 自动部署

每次推送到 `main` 分支时，GitHub Actions 会自动：

1. 检出代码
2. 安装依赖
3. 构建项目
4. 部署到 GitHub Pages

### 手动部署

如果需要手动触发部署：

1. 进入仓库的 Actions 页面
2. 选择 "Deploy to GitHub Pages" 工作流
3. 点击 "Run workflow"
4. 选择 `main` 分支
5. 点击 "Run workflow" 按钮

### 本地构建测试

在推送前，可以本地测试构建：

```bash
# 构建项目
npm run build

# 测试构建产物
npm run test:build

# 预览构建结果
npm run preview
```

### 访问地址

部署成功后，可以通过以下地址访问：

```
https://re0project.github.io/DarkSouls-Tome/
```

### 自定义域名（可选）

如果要使用自定义域名：

1. 在仓库根目录创建 `public/CNAME` 文件
2. 文件内容为你的域名，例如：`darksouls.example.com`
3. 在域名提供商处配置 DNS：
   - 类型：CNAME
   - 名称：www（或其他子域名）
   - 值：re0project.github.io
4. 推送代码，等待部署完成

## 其他部署方式

### Vercel 部署

1. 导入 GitHub 仓库到 Vercel
2. 构建命令：`npm run build`
3. 输出目录：`dist`
4. 环境变量：无需配置
5. 点击部署

### Netlify 部署

1. 连接 GitHub 仓库
2. 构建命令：`npm run build`
3. 发布目录：`dist`
4. 点击部署

### 服务器部署

如果要部署到自己的服务器：

```bash
# 1. 构建项目
npm run build

# 2. 将 dist 目录上传到服务器
scp -r dist/* user@server:/var/www/html/

# 3. 配置 Nginx
server {
    listen 80;
    server_name your-domain.com;
    root /var/www/html;
    index index.html;

    location / {
        try_files $uri $uri/ /index.html;
    }
}

# 4. 重启 Nginx
sudo systemctl restart nginx
```

## 构建配置

### 环境变量

项目支持以下环境变量：

- `NODE_ENV`: 环境模式（development/production）
- `BASE_URL`: 应用基础路径（默认：`/DarkSouls-Tome/`）

### 修改基础路径

如果部署到根路径或其他路径，修改 `vite.config.ts`：

```typescript
export default defineConfig({
  base: '/', // 根路径
  // 或
  base: '/your-path/', // 自定义路径
  // ...
})
```

## 故障排查

### 页面空白

1. 检查浏览器控制台错误
2. 确认 `base` 路径配置正确
3. 检查资源路径是否正确

### 404 错误

1. 确认 GitHub Pages 已启用
2. 检查仓库名称是否正确
3. 等待几分钟让部署生效

### 样式丢失

1. 检查 CSS 文件是否正确加载
2. 确认 `base` 路径包含在资源路径中
3. 清除浏览器缓存

### 图片不显示

1. 确认图片文件在 `public/icons` 目录
2. 检查图片路径是否正确
3. 确认 `.nojekyll` 文件存在

## 性能优化建议

### CDN 加速

使用 CDN 加速静态资源：

1. 将构建产物上传到 CDN
2. 修改资源引用路径
3. 配置 CDN 缓存策略

### 缓存策略

配置合适的缓存头：

```nginx
# HTML 文件不缓存
location ~* \.html$ {
    add_header Cache-Control "no-cache, no-store, must-revalidate";
}

# JS/CSS 文件长期缓存
location ~* \.(js|css)$ {
    add_header Cache-Control "public, max-age=31536000, immutable";
}

# 图片文件缓存
location ~* \.(webp|png|jpg|jpeg)$ {
    add_header Cache-Control "public, max-age=2592000";
}
```

### Gzip 压缩

启用 Gzip 压缩减少传输大小：

```nginx
gzip on;
gzip_types text/plain text/css application/json application/javascript text/xml application/xml application/xml+rss text/javascript;
gzip_min_length 1000;
```

```html

<script async src="https://www.googletagmanager.com/gtag/js?id=GA_MEASUREMENT_ID"></script>

  window.dataLayer = window.dataLayer || [];

  gtag('config', 'GA_MEASUREMENT_ID');
</script>
```
