### 2. 创建改进版导航组件

#### 设计理念
参考艾尔登法环WIKI的设计，但不照抄，加入以下特色：
- 支持游戏版本切换
- 更现代的布局
- 更好的移动端体验
- 保持黑魂风格

#### 组件结构

**ImprovedNavigation.vue**

```
┌─────────────────────────────────────────┐
│ Logo              [黑魂1][黑魂2][黑魂3] │  ← 顶部栏
├─────────────────────────────────────────┤
│ ⚔️武器 🛡️防具 💍戒指 📦物品 ✨法术 💬对话 ★收藏 │  ← 分类导航
└─────────────────────────────────────────┘
```

#### 核心功能

1. **Logo 区域**
   - 中文名称：黑魂数据库
   - 英文副标题：Dark Souls DB
   - 点击返回首页

2. **游戏切换器**
   - 三个按钮：黑魂1、黑魂2、黑魂3
   - 当前游戏高亮显示
   - 点击切换游戏版本
   - 自动跳转到对应游戏的相同分类

3. **分类导航**
   - 6个物品分类 + 收藏
   - 图标 + 中文名 + 英文名
   - 当前页面高亮
   - 悬停效果

4. **移动端菜单**
   - 汉堡菜单按钮
   - 下拉式菜单
   - 游戏切换器
   - 分类列表

#### 2. 游戏切换逻辑
```typescript
const currentGame = computed(() => {
  const match = route.path.match(/\/ds(\d)/);
  return match ? Number(match[1]) : 1;
});

const switchGame = (gameId: number) => {
  const currentType = route.params.type;
  if (currentType) {
    router.push(`/ds${gameId}/${currentType}`);
  }
};
```

## 对比分析

### 原 ClassicNavigation vs 新 ImprovedNavigation

| 特性 | ClassicNavigation | ImprovedNavigation |
|------|-------------------|-------------------|
| 布局 | 4行固定布局 | 2行动态布局 |
| 游戏切换 | 每行一个游戏 | 顶部切换器 |
| 分类导航 | 每行6个链接 | 单行7个链接 |
| 图标 | 无 | 有（emoji） |
| 移动端 | 缩小字体 | 独立菜单 |
| 空间利用 | 150px 高度 | 120px 高度 |
| 交互体验 | 基础 | 增强 |

### 优势

#### ImprovedNavigation 的优势
1. **更好的空间利用**
   - 减少30px高度
   - 更紧凑的布局
   - 更多内容展示空间

2. **更好的用户体验**
   - 游戏切换更直观
   - 分类导航更清晰
   - 移动端体验更好

3. **更现代的设计**
   - 图标增强识别
   - 毛玻璃效果
   - 更好的视觉层次

4. **更灵活的扩展**
   - 易于添加新分类
   - 易于添加新功能
   - 易于自定义样式

#### ClassicNavigation 的优势
1. **更接近原版**
   - 完全复刻原版布局
   - 保持原版风格

2. **更简单的实现**
   - 无状态管理
   - 无复杂逻辑
